# Profile Lock Finalization Contract

## Goal

Convert replay batch results into a deterministic per-scenario lock decision and persist locked profile artifacts.

## Inputs

1. `replay_report.json` generated by:

```bash
PYTHONPATH=src python3 /Users/seongcheoljeong/Documents/Codex_test/scripts/run_moving_target_replay_batch.py \
  --manifest-json /path/to/replay_manifest.json \
  --output-json /path/to/replay_report.json
```

2. Optional lock policy overrides:

- `min_pass_rate` (`0.0 ~ 1.0`)
- `max_case_fail_count` (`>=0`)
- `require_motion_defaults_enabled` (`True/False`)

## Lock Finalization CLI

```bash
PYTHONPATH=src python3 /Users/seongcheoljeong/Documents/Codex_test/scripts/finalize_scenario_profile_lock.py \
  --replay-report-json /path/to/replay_report.json \
  --output-lock-json /path/to/profile_lock_report.json \
  --output-locked-profile-dir /path/to/locked_profiles \
  --require-motion-defaults-enabled
```

Exit codes:

- `0`: all scenarios are locked (or `--allow-unlocked` used)
- `2`: one or more scenarios are unlocked

## Output

`profile_lock_report.json` contains:

- `policy`
- `overall_lock_pass`
- `summary` (`case_count`, `locked_count`, `unlocked_count`)
- `cases[]`:
  - `scenario_id`
  - `candidate_count`, `pass_count`, `fail_count`, `pass_rate`
  - `motion_defaults_enabled`
  - `lock_pass`
  - `lock_reasons[]`

When `--output-locked-profile-dir` is set:

- writes `<scenario_id>.locked.json`
- each file includes `profile_lock` section:
  - `locked`
  - `locked_utc`
  - `policy`
  - replay summary fields
  - `lock_reasons[]`
